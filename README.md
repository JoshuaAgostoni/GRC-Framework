# GRC & Incident Response Framework

##  Project Overview:
This project represents the transition from technical implementation to **Governance, Risk, and Compliance (GRC)** and **Incident Response (IR)** strategy. It demonstrates the ability to translate technical security findings into professional documentation, risk management registers, and Standard Operating Procedures (SOPs). 

The goal of this lab was to establish an enterprise-level security governance framework that ensures technical controls (like those in my AWS and Active Directory labs) are backed by documented authority and repeatable processes.

---

## Framework Components:

### 1. Enterprise Risk Register:
* **File:** `Enterprise-Risk-Register.csv`
* **Description:** A comprehensive log used to identify, assess, and prioritize organizational threats such as **Phishing**, **Ransomware**, and **Unauthorized Cloud Access**.
* **Key Fields:** Risk Owners, Impact/Probability levels, and specific mitigation actions mapped to technical controls.

### 2. Cloud Security Baseline (SOP-Cloud-01)
* **File:** `SOP-Cloud-01_Security-Baseline.md`
* **Description:** A standardized operating procedure defining the mandatory security configurations for cloud environments.
* **Focus Areas:** Identity and Access Management (IAM), Data Encryption (At-rest/In-transit), and Network Security Logging.

### 3. Incident Response: Triage & Analysis (SOP-IR-02)
* **File:** `SOP-IR-02_Triage-and-Analysis.md`
* **Description:** A formal framework for responding to security alerts, ensuring a repeatable and audited process for the SOC team.
* **Focus Areas:** Initial classification, evidence preservation, and escalation paths for high-priority incidents.

---

## Security Principles Demonstrated:
* **Risk Mitigation:** Prioritizing security efforts based on business impact and threat likelihood.
* **Standardization:** Creating "Golden Standards" for cloud deployment to reduce configuration drift.
* **Incident Readiness:** Moving beyond "detection" to a structured, documented response phase.
* **Compliance Mapping:** Ensuring technical actions align with industry best practices and organizational risk tolerance.
